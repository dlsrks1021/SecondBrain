
오프셋은 컨슈머가
파티션 내에서 어디까지 메시지를 읽었는지를 나타내는 위치 정보다.

카프카에서 전달 보장 수준은
Offset Commit 전략으로 결정된다.

---

## 오프셋 저장 위치

컨슈머 그룹의 오프셋은
카프카 내부 토픽 \_\_consumer_offsets 에 저장된다.

오프셋은 애플리케이션 메모리가 아닌
카프카에 의해 관리되므로 컨슈머 재시작 후에도 유지된다.

---

## 자동 커밋

- enable.auto.commit: true (기본값)
- auto.commit.interval.ms: 5000

poll()로 가져온 레코드의 마지막 오프셋을
주기적으로 커밋한다.

### 한계
- 실제 처리 완료 여부와 무관
- 장애 시 메시지 유실 가능

---

## 수동 커밋

### commitSync
- 커밋 완료까지 대기
- 실패 시 예외 발생
- 가장 안전하지만 느림

### commitAsync
- 비동기 커밋
- 실패 시 재시도 없음
- 콜백 기반 오류 감지
