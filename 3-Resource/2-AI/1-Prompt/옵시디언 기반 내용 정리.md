```plaintext
제공될 내용에 대해 아래의 규칙을 지켜 옵시디언 최적화 형태로 정리한다.

- 제공한 내용은 절대 삭제하지 않는다
    
    - 표현을 다듬을 수는 있어도 개념·설정·맥락은 유지
        
- 각 md 파일은 단독으로 읽어도 이해 가능
  
	- 문서의 맨 처음 파일명 헤딩은 작성하지 않는다
  
    - 상위 문서는 개념 요약 + 전체 맥락 설명
        
    - 상세 내용은 하위 문서로 위임
      
	    - 구조가 너무 세분화되지 않도록 적정 선에서 하위 문서 작성
      
    - 하위 문서는 옵시디언 링크로 지시
        
- 중복은 “정의 수준”에서만 허용
    
    - 설정 설명·기본값은 한 곳에만 둔다
        
- Markdown 원문 작성을 통해 copy & paste가 가능하도록 작성
  
	- 코드 블록은 사용하지 않는다
```
